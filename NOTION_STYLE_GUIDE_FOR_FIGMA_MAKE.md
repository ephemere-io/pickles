## 5. アニメーション・トランジション

Notionのアニメーションは**極めて控えめで高速**。装飾ではなく応答性を強化する目的。

### 5.1 トランジション一覧

| コンテキスト | 時間 | イージング |
|---|---|---|
| デフォルトUI | `120ms` | `ease-in` |
| 背景色変化 | `100ms` | `ease-out` |
| ボーダー色変化 | `100ms` | `ease-in` |
| 不透明度フェードイン | `400ms` | `ease-in` |
| 不透明度フェードアウト | `400ms` | `ease-out` |
| サイドバー不透明度 | `20ms` | `ease-in` |
| サイドバー幅変化 | `300ms` | `ease-in` |
| ドロップダウンスライドダウン | `400ms` | `cubic-bezier(0.16, 1, 0.3, 1)` |
| スピナー回転 | `600ms` | `linear infinite` |
| モーダル開く | `200-300ms` | `ease-out`, `scale(0.95 → 1.0)` + `opacity(0 → 1)` |
| モーダル閉じる | `100-150ms` | 高速（閉じは開くより速く） |
| ページ遷移 | `400ms` | `cubic-bezier(0.61, 1, 0.88, 1)`, `y: 8 → 0` + `opacity` |

### 5.2 ホバーツールチップ

- 表示遅延: `100ms`（ほぼ瞬時）
- 配置: `bottom` がデフォルト（top / left / right も可）

## 7. UIコンポーネント仕様

### 7.1 サイドバー

```
構造:
├── ワークスペース名 + アバター
├── 検索バー (30px高, Cmd+K でも開く)
├── メインナビ (131px高)
│   ├── 検索
│   ├── Notion AI
│   ├── ホーム
│   └── 受信トレイ
├── お気に入り (折りたたみ可能, 30pxヘッダー)
├── チームスペース (折りたたみ可能)
├── 共有 (折りたたみ可能)
├── プライベート (折りたたみ可能)
│   └── ページツリー (アコーディオン/ツリーナビ)
│       └── ネストされた展開/折りたたみトグル
└── ユーティリティ (設定, ゴミ箱)

ビジュアル:
- 各アイテム: 絵文字アイコン + ページ名
- ホバー時: 右側に「...」メニューと「+」ボタンが出現
- セクション間: 6px ギャップ
- フォント: システムフォント, medium ウェイト, ウォームグレー
- 折りたたみ: Cmd+\ で切替、<<ボタンで非表示、左端ホバーで一時表示
```

### 7.2 ページヘッダー

```
構造:
├── パンくずリスト (最大3セグメント表示, 深いネストは中間省略)
├── 右上: 共有 / コメント / 更新 / お気に入り / ... メニュー
├── カバー画像 (任意, 1500x600px推奨)
├── ページアイコン (絵文字 or カスタム画像, カバー画像と重なる位置)
└── ページタイトル ("無題" プレースホルダー, 40px, font-weight: 700)

下部にプロパティ表示:
├── ラベル (グレーテキスト, 左寄せ)
└── 値 (右寄せ, インライン編集可能)
```

### 7.3 ブロックエディタ

```
すべてのコンテンツはブロック:
- テキスト、画像、埋め込み、データベース、区切り線、コールアウトなど

各ブロック:
├── 左マージン: ホバー時に「+」ボタンと「⋮⋮」ドラッグハンドル (#C3C2C0) が出現
├── ブロック本体: 全幅、contentEditable
├── プレースホルダー: "Type '/' for commands" (空ブロック時、フォーカスで消える)
└── ブロック間マージン: 1-2px (密な配置)

選択状態:
- テキスト選択: 青いハイライトオーバーレイ
- ブロック選択 (Esc): 青い背景ハイライト
- 範囲選択: Shift+Click で現在〜クリック先まで全ブロック選択
```

### 7.4 フローティングフォーマットツールバー

```
テキスト選択時に出現:
├── Bold | Italic | Underline | Strikethrough | Code
├── Link | テキストカラー | Comment
└── AI オプション

配置: 選択テキストの上または下 (ビューポート内に収まるよう自動調整)
スタイル: ダークグレー背景、白テキスト、コンパクト
```

### 7.5 スラッシュコマンドメニュー

```
「/」入力で即座に表示:
├── 検索フィルター (リアルタイムファジー検索)
├── カテゴリ分類:
│   ├── Basic Blocks (テキスト, 見出し, リスト, トグル, 引用等)
│   ├── Inline (メンション, 日付, 数式)
│   ├── Database (テーブル, ボード, カレンダー, リスト, ギャラリー, タイムライン)
│   ├── Media (画像, 動画, オーディオ, ファイル, ブックマーク)
│   ├── Embeds (外部サービス埋め込み)
│   └── Advanced (目次, ブレッドクラム, 数式ブロック)
└── 各コマンド: アイコン + 名前 + 説明 + キーボードショートカット

操作: ↑↓ でナビ、Enter で選択、Esc で閉じ
配置: カーソル位置の下（上に十分なスペースがない場合は上）

## 8. UXインタラクション仕様

### 8.1 インライン編集

- クリックした場所がそのまま編集可能（モーダルやフォーム遷移なし）
- `contentEditable` を使用したリッチテキスト編集
- 空ブロックにプレースホルダーテキスト表示、フォーカスで消える
- Esc でブロック選択モード ⇔ Enter でテキスト編集モード

### 8.2 ドラッグ & ドロップ

```
操作フロー:
1. ブロックホバー → 左マージンに「⋮⋮」ドラッグハンドル出現
2. クリック+ホールド → ドラッグ開始、カーソルがgrab/moveに変化
3. ドラッグ中:
   - ブロック間に青いガイドライン (2-3px, Notion blue #2383E2) 表示
   - 全幅の水平線 → ブロック挿入位置
   - 部分幅の垂直線 → カラム作成
4. ドロップ → ブロック移動完了

追加操作:
- Option/Alt + ドラッグ = ブロック複製（移動ではなくコピー）
- ハンドルクリック（ドラッグなし）= コンテキストメニュー表示
  (Turn into / Color / Copy link / Duplicate / Move to / Comment / Delete)
```

### 8.3 ブロック操作キーボードショートカット

| アクション | ショートカット |
|---|---|
| ブロック複製 | `Cmd/Ctrl + D` |
| ブロック上下移動 | `Cmd/Ctrl + Shift + ↑/↓` |
| ブロックアクションメニュー | `Cmd/Ctrl + /` |

## 9. 空状態・ローディング・オンボーディング

### 9.1 空状態 (Empty State)

```
新規ページ:
- タイトル: "無題" プレースホルダー
- 本文: "Press Enter to continue with an empty page, or pick a template"
- クイックアクションボタン: テンプレート、インポート等
- デザイン: シンプルなモノクロイラスト、控えめで暖かい

データベース空フィールド:
- "Empty" グレーアウトプレースホルダー
- フォーカスで消える

検索結果なし:
- クリーンなメッセージ + 代替案の提案
```

### 9.2 ローディング

- **スケルトンスクリーンは基本的に使わない**（データがスケルトン表示より速くロードされるケースが多いため）
- **楽観的更新（Optimistic Update）**: ローカルで即座にレンダリング、バックグラウンドで同期
- ローディングインジケーターは本当に遅い場合のみ表示

### 9.3 オンボーディング

- パーソナライゼーション調査（ドロップダウン式、自由入力ではない）
- プロフィール作成（名前 + 写真）→ オーナーシップ感の醸成
- **漸進的開示（Progressive Disclosure）**: 機能を適切なタイミングで紹介
- チェックリスト型学習: 実際のコンテンツを操作しながら学ぶ

## 11. デザイン原則まとめ

Figma Makeへの指示に含めるべきNotionの設計哲学:

### 11.1 制約付きの柔軟性 (Constrained Flexibility)
- 10色のみ。フォントスタイルはページ単位で1つ。「醜くならない」設計。
- カスタマイズ手段は絵文字アイコンとカバー画像が中心。

### 11.2 ホバーで現れるUI (Progressive Disclosure)
- 操作要素は普段隠れている。ホバー時にのみ出現。
- ドラッグハンドル、+ボタン、...メニュー、アクションボタン。
- デフォルトはコンテンツに集中させる。

### 11.3 キーボードファースト (Keyboard-First)
- スラッシュコマンド、Markdownショートカット、コマンドパレット。
- ショートカットをUIに表示し、ユーザーが自然に学習する設計。

### 11.4 ブロックベース (Everything is a Block)
- すべてのコンテンツが独立したブロック。移動、変換、複製が自由。
- 全幅デフォルト。ドラッグでカラム作成。

### 11.5 空間的メニュー配置 (Spatial Menu Placement)
- アプリ設定 → サイドバー左ナビ
- ページ設定 → 右上コーナー
- ブロック設定 → ブロック横のインライン
- ユーザーを圧倒しない、コンテキストに沿った配置。

### 11.6 ミニマリズム
- 装飾を排除。ウォームグレー（純黒を避ける）。広いホワイトスペース。
- 「良いデザインはそれ自体に注意を向けさせない」

## 14. Figma Make への具体的な指示例文

以下をコピーしてFigma Makeに渡すことで、Notionスタイルを正確に再現できます:

```
このアプリケーションのUIをNotionに限りなく近づけてください。以下の仕様に厳密に従ってください。

【カラー】
- プライマリテキスト: #37352F (純粋な黒は使わない、ウォームグレー)
- セカンダリテキスト: rgba(55, 53, 47, 0.6)
- プレースホルダー: rgba(55, 53, 47, 0.4)
- ページ背景: #FFFFFF
- サイドバー背景: #F7F6F3
- ホバー背景: rgba(55, 53, 47, 0.08)
- ボーダー: rgba(55, 53, 47, 0.09) — 極めて薄い
- アクセント（リンク・選択）: rgb(46, 170, 220)
- テキスト選択ハイライト: rgba(45, 170, 219, 0.3)

【タイポグラフィ】
- フォント: Inter (フォールバック: system-ui, -apple-system, sans-serif)
- ページタイトル: 40px, weight 700, line-height 1.2
- H1: 30px, weight 600, line-height 1.3
- H2: 24px, weight 600
- H3: 20px, weight 600
- 本文: 16px, weight 400, line-height 1.5
- UIテキスト: 14px, weight 500

【レイアウト】
- 8pxグリッドベース
- コンテンツ幅: 720px中央揃え (左右 calc(min(96px, 8vw)) パディング)
- サイドバー: 左固定240px幅
- ヘッダー: 45px高さ

【ボーダー・シャドウ】
- 角丸: 3px（ブロック）、6px（ボタン）
- カードシャドウ: rgba(15,15,15,0.1) 0 0 0 1px, rgba(15,15,15,0.1) 0 2px 4px
- モーダルシャドウ: rgba(15,15,15,0.05) 0 0 0 1px, rgba(15,15,15,0.1) 0 5px 10px, rgba(15,15,15,0.2) 0 15px 40px

- ホバー背景色変化: 100ms ease-out
- モーダル: scale(0.95→1.0) + opacity(0→1), 200ms ease-out

【デザイン原則】
- 徹底的なミニマリズム。装飾排除。コンテンツに集中。
- ウォームグレートーン（純黒・純白を避ける）
- 広いホワイトスペース
- ブロックベース構造（すべてのコンテンツは独立ブロック）
- 操作UIは必要時のみ出現（ホバーで表示）

参考Figmaファイル:
- Unofficial Notion Design System v1.1 (figma.com/community/file/877573866872969565)
- Notion UI Kit Recreated (figma.com/community/file/1279161103014638492)
